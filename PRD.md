> **한 줄 요약**: 집중 시간 동안 귀여운 눈알 캐릭터가 주기적으로 팝업되어 "지켜보는" macOS 메뉴바 뽀모도로 타이머. GitHub 공개 + Homebrew 배포가 최종 목표.

## 1. 개요 & 문제 정의

### 문제
기존 뽀모도로 타이머 앱은 **수동적(passive)** 이다. 사용자가 타이머를 시작하고 나면 그 존재를 잊어버리고, 어느새 유튜브·SNS·딴 창으로 빠져 있다. 타이머는 조용히 돌아가지만 집중을 "유지"시켜 주지는 못한다.

### 해결
집중 세션이 진행되는 동안 **귀여운 눈알 캐릭터가 주기적으로 화면에 나타나 잠깐 지켜보고 사라진다.** 이 가벼운 존재감이 "누가 보고 있다"는 부드러운 사회적 압박(soft accountability)을 만들어, 위협적이지 않으면서도 사용자를 화면 앞으로 다시 끌어온다. 감시라기보단 응원에 가까운, 밉지 않은 감시자.

### 차별점
- 대부분의 뽀모도로 앱: 타이머 + 알림뿐 → 잊혀짐
- 본 제품: **능동적 체크인 캐릭터** → 존재감으로 집중 유도
- 네이티브 macOS 앱: 가볍고(수십 MB), 메뉴바에 자연스럽게 녹아듦

### 타겟 사용자
- 혼자 일하는 개발자 / 학생 / 재택근무자
- macOS를 주력으로 쓰며, 가볍고 방해되지 않는 도구를 선호하는 사람
- 뽀모도로 기법을 쓰지만 "시작하고 잊어버리는" 문제를 겪는 사람

## 2. 핵심 컨셉 — 주기적 체크인 캐릭터

제품의 심장. 이 기능이 여느 뽀모도로 앱과의 유일하고 결정적인 차별점이다.

### 동작 방식
1. **집중 세션 진행 중**, 캐릭터가 **메뉴바 아이콘 아래에서 아래로 쑥 튀어나온다(peek)**. 잠시 지켜본 뒤 다시 메뉴바 뒤로 숨는다. (귀 없음 — "튀어나오는 것"이 핵심)
2. 등장한 캐릭터는 **3~5초간 눈동자로 마우스 커서를 따라가며** "지켜본" 뒤 사라진다.
3. **등장 시점은 무작위(N ± X)** — 기준 주기 N에 대해 X는 균등분포 `|X| < 0.5N`이라 실제 간격이 `(0.5N, 1.5N)`에서 흔들린다. 유저가 언제 나올지 예측하지 못해 감시 효과가 유지된다.

### UX 원칙 — 방해하지 않기 (Non-intrusive)
캐릭터는 절대 작업을 방해해선 안 된다. 짜증나는 순간 사용자는 앱을 지운다.
- **클릭 통과(click-through)**: 캐릭터 창은 마우스 이벤트를 아래 창으로 그대로 흘려보낸다.
- **포커스 뺏지 않음**: non-activating 패널 → 타이핑 중이던 앱의 커서/포커스를 절대 가로채지 않는다.
- **항상 위(floating)**: 다른 창 위에 떠 있되, 메뉴바 아래 작은 영역만 차지한다.
- **끌 수 있음**: 체크인 주기를 "끄기"로 설정하면 순수 타이머로만 동작.

> **휴식 시간은 예외** — 집중 중엔 방해하지 않지만, 휴식이 시작되면 의도적으로 작업을 멈추게 하는 전체화면 오버레이를 띄운다(§3.4 참고, Flow 앱 방식).

### 비주얼 에셋 방침
**모든 캐릭터는 코드로 그린다 (SwiftUI `Canvas` / `Shape` / `Path`).**
- 저작권/라이선스 이슈 없음 → GitHub 공개·재배포에 깔끔
- 눈 깜빡임, 커서 추적, 표정 변화 등 인터랙션을 코드로 자연스럽게 구현
- 벡터라 모든 해상도에서 선명, 번들 용량 추가 없음

## 3. v1 기능 명세

첫 공개 릴리즈(v1)는 **타이머 코어 + 감시 캐릭터 + 휴식 오버레이 + 사운드**에 집중한다. 아래 Non-goals는 v2 이후로 미룬다.

### 3.1 타이머 (Core)
- 세 가지 세션: **집중(Focus)** / **짧은 휴식(Short Break)** / **긴 휴식(Long Break)**
- 기본값: 25분 / 5분 / 15분, 사용자가 분 단위로 조절 가능
- 긴 휴식 주기: 집중 세션 N회(기본 4회)마다 긴 휴식
- 조작: 시작 / 일시정지 / 리셋 / 스킵(다음 세션으로)

### 3.2 메뉴바 (MenuBarExtra)
- 메뉴바에 **남은 시간 카운트다운** 상시 표시 (예: `24:37`)
- 현재 세션 종류를 아이콘/색으로 구분
- 클릭 시 컨트롤 팝오버: 시작·정지 버튼, 세션 전환, 설정 진입

### 3.3 체크인 팝업 (핵심 차별 기능)
- 주기 설정: **끄기 / 3분 / 5분 / 10분** (실제 등장은 §2의 무작위 N±X)
- 메뉴바 아이콘 아래에서 아래로 튀어나오는(peek) 애니메이션, 커서 추적 동작
- 집중 세션 중에만 등장 (휴식 중엔 등장 안 함)
- 위치: 메뉴바 아이콘 x에 맞춤. MenuBarExtra가 아이콘 좌표를 직접 노출하지 않아 상태바 창 프레임을 탐색하고, 못 찾으면 우측 상단으로 폴백

### 3.4 휴식 오버레이 (Flow 앱 방식)
집중이 끝나면 휴식을 **자동 시작**하고 화면 전체를 덮는 오버레이로 작업을 멈추게 한다.
- **휴식 중(resting)**: 프로스트 블러 + 쉬는 얼굴 캐릭터 + 큰 카운트다운. "가리기 닫기"(× 또는 Esc)로 언제든 닫을 수 있고, 닫아도 휴식은 메뉴바에서 계속 카운트다운. "지금 집중 시작"으로 휴식을 건너뛸 수도 있음
- **휴식 종료(ready)**: 휴식이 끝나면 **"집중 다시 시작" 프롬프트가 알아서 팝업** — 사용자가 준비되면 눌러 다음 집중을 재개(자동 시작 아님)
- 모든 디스플레이를 덮음, 접근성 "동작 줄이기" 존중

### 3.5 알림 & 사운드
- 세션 전환 시 macOS 로컬 알림 (`UserNotifications`)
- 세션 완료 사운드 (on/off 토글, 시스템 사운드 활용)

> macOS 집중 모드(Focus/DND) 자동 연동은 **v1에서 제외**(공식 API 제약 + 휴식 오버레이가 이미 강한 개입을 제공). 필요 시 v2에서 Shortcuts 연동으로 검토.

### 3.6 편의
- 로그인 시 자동 시작 옵션 (`SMAppService`)
- Dock 아이콘 없이 메뉴바에만 상주 (`LSUIElement`)

### Non-goals (v1 제외 → v2 후보)
- 집중 통계 / 세션 기록 대시보드
- 캐릭터·눈알 스타일 커스터마이징 (여러 캐릭터 선택)
- 태스크/할 일 연동
- iOS / Windows / Linux 등 타 플랫폼
- "딴짓 감지"(활성 앱 추적) 기반 능동 반응 — v2 후보로만 기록

## 4. 기술 설계 개요

### 스택
- **Swift 5.9+ / SwiftUI**, 최소 타겟 **macOS 13** (`MenuBarExtra` 사용을 위해)
- `LSUIElement = YES` → Dock 아이콘 없이 메뉴바 상주

### 아키텍처
- **`TimerEngine`** (단일 상태 머신): `idle → focus → shortBreak / longBreak` 전환, 남은 시간·세션 카운트 관리. `ObservableObject`로 UI에 바인딩.
- **메뉴바 UI**: `MenuBarExtra` + 팝오버(SwiftUI View)
- **체크인 창**: non-activating `NSPanel`
  - `.styleMask = [.nonactivatingPanel]`, `.level = .floating`
  - `ignoresMouseEvents = true` (클릭 통과)
  - `collectionBehavior`에 `.canJoinAllSpaces`, `.stationary` 등 지정
  - 내부는 SwiftUI 캐릭터 View 호스팅 (`NSHostingView`)
- **캐릭터 렌더링**: SwiftUI `Canvas`/`Shape`, 커서 위치는 `NSEvent.mouseLocation` 폴링 또는 글로벌 모니터로 추적
- **알림**: `UserNotifications` 프레임워크
- **자동 시작**: `SMAppService.mainApp` (macOS 13+)

### 산출물
- `.app` 번들 → **GitHub Releases**에 `.dmg` 또는 `.zip`으로 업로드

## 5. 배포 계획 (GitHub + Homebrew)

### 5.1 GitHub 공개
- MIT 라이선스 공개 저장소
- README(**영문**): 스크린샷, **캐릭터 동작 GIF 데모**(가장 중요한 세일즈 포인트), 설치법
- 이슈/PR 템플릿, 기본 CI

### 5.2 Homebrew 배포 (2단계 전략)
1. **1단계 — 개인 Tap**: `homebrew-<앱이름>` 저장소를 만들어 Cask 등록.
   → 사용자는 `brew install --cask <owner>/<tap>/<app>`으로 설치.
   → 심사 없이 즉시 배포 가능, 초기 사용자 확보용.
2. **2단계 — 공식 Cask 제출**: 스타·사용자가 쌓이면 `homebrew/homebrew-cask` 본진에 PR.
   → notability(지명도) 요건 존재 → 1단계에서 사용자 기반을 먼저 만든 뒤 진행.

### 5.3 릴리즈 자동화
- **GitHub Actions**: 태그 push → 빌드 → `.dmg` 생성 → Release 자동 업로드 (v1.x 중 도입)
- Cask의 `sha256`·버전 자동 갱신 스크립트 고려

## 7. Open Questions (구현 전 결정 필요)

1. **앱 이름** — 후보: *Peekaboo*, *Peekodoro*, *EyeModoro*, *Watchmodoro* 등. (`.app` 이름·번들 ID·Tap 이름·브랜딩에 영향)
2. **코드 서명 / 공증(Notarization)** — Apple Developer Program($99/년)에 가입해 Developer ID로 서명·공증할지, 아니면 미서명 배포 + Gatekeeper 우회(`우클릭 → 열기`) 안내로 갈지. Homebrew Cask는 미서명도 허용하나 사용자 경험 저하.
3. **macOS Focus(집중 모드) 자동화 방식** — 공식 API가 제한적. Shortcuts 앱 연동 / `Intents` / 단순 안내형 중 무엇으로 갈지 M3에서 결정.
4. **체크인 캐릭터의 "딴짓 감지" 반응**(활성 앱이 유튜브·SNS면 캐릭터가 화내는 등) — 매력적이나 프라이버시·복잡도 이슈. **v2 후보로만 기록**, v1 범위 밖.
